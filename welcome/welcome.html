Contexta Performance Recorder

Record browser sessions and generate production-ready JMeter performance test scripts — with automatic correlation detection, field classification, and assertion generation.

Quick Start

Click the Contexta icon in your browser toolbar

The recorder popup opens. Set your journey code (e.g. "Login", "Booking") and recording mode.

Click "Start Recording" and use the website normally

Navigate through the journey you want to performance test. Fill in forms, click buttons, follow the flow.

Click "Stop" when you've completed the journey

The recorder analyses your session — detecting correlations, classifying form fields, and generating assertions.

Download your JMX script

Choose Fragment mode (for PerftestFramework) or Standalone (run directly in JMeter). Also export HAR, JSON, or CSV test data.

Recording Modes

Auto (recommended)

Automatically creates a new transaction on each page navigation. Best for standard web flows — login, forms, multi-step wizards.

Transaction

You manually mark transaction boundaries by clicking "New Transaction" in the popup. Use this when you want precise control over step grouping.

Full

Records everything as a single block. Split into transactions later. Useful for exploratory recording where you're not sure about the boundaries yet.

Export Formats

JMX — Fragment Mode

Generates a TestFragmentController, designed for use with an IncludeController in a test orchestrator like PerftestFramework. Contains transaction controllers, HTTP samplers, extractors, assertions, and think time — but no thread group or test plan wrapper.

JMX — Standalone Mode

Generates a complete TestPlan with thread group, HTTP request defaults, cookie/cache managers, and a header manager. Ready to open and run directly in JMeter.

HAR

Standard HTTP Archive format (HAR 1.2). Compatible with browser DevTools, Charles Proxy, Fiddler, and other tools. Good for sharing recordings with developers.

JSON

The complete recording session including all analysis data: correlations, field classifications, assertions, fingerprints, and page responses. Use this for debugging or feeding into other tools.

CSV

A test data template with column headers derived from the form fields found during recording, plus one row of sample data. Use as a starting point for your test data files.

FAQ

What gets filtered out?

Static resources (CSS, JS, images, fonts), third-party analytics (Google Analytics, Hotjar, etc.), and browser-internal requests are automatically excluded. You can customise excluded domains in the extension options.

Does it work with Single Page Applications?

Yes. The recorder captures XHR and Fetch requests alongside page navigations. For SPAs, use "Transaction" mode so you can mark step boundaries manually.

Can I record behind a login?

Yes. Start recording before you log in, and the recorder captures the entire authentication flow including session cookies and CSRF tokens. These are automatically correlated in the JMX output.

How do correlations work?

The recorder scans page HTML for dynamic values (hidden inputs, meta tags, embedded JSON) and tracks where those values appear in subsequent POST requests. It then generates JMeter RegexExtractors or JSONPostProcessors to extract them at runtime.

What about think time?

A UniformRandomTimer is added to the JMX with configurable min/max delays (default 500ms-3000ms). In PerftestFramework mode, these use JMeter properties so you can override them from the pipeline.

Do I need a Contexta account?

No. The recorder works completely standalone — record, analyse, and download scripts with no account needed. PerfOps Studio is a separate product you can sign into directly at perfops.studio to build test plans visually and execute distributed load tests.
