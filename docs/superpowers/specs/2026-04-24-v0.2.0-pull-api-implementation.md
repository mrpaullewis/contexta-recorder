# Implementation Spec — Contexta Recorder v0.2.0 Pull API

**Target:** Contexta Recorder Chrome extension v0.2.0
**Audience:** Whoever is coding the extension side
**Status:** Ready to implement
**Companion doc:** `docs/RECORDER-PULL-API.md` (the public API contract — read that first if you haven't)

Assumption: the Studio side has a POST endpoint ready to receive the imported recording. The extension's job is to expose its stored recordings to any whitelisted origin via `chrome.runtime.sendMessage`.

## 2. Manifest changes

```jsonc
// manifest.json — add this top-level key
"externally_connectable": {
  "matches": [
    "https://*.perfops.studio/*",
    "https://studio.perfops.studio/*",
    "http://localhost:*/*",        // dev — keep or strip before publishing
    "http://127.0.0.1:*/*"         // dev — keep or strip before publishing
  ]
}
```

**CWS hygiene rule:** strip the localhost entries before zipping for production submission. Chrome Web Store reviewers may challenge wildcard localhost patterns. Keep them in a separate `manifest.dev.json` or use a build-step substitution. If using unpacked dev extension only, they can stay.

**Decision to make:** whether `https://*.perfops.studio/*` is broad enough or if we should hard-list each Studio subdomain. Broad pattern = less maintenance, narrower = more CWS-friendly. Recommend broad for v0.2.0.

## 3. Service worker message handler — concrete implementation

Add to `background/service-worker.js`. Place after the existing `chrome.runtime.onMessage` listener (internal messages) so the two listeners are side by side.

```javascript
// ── External pull API (v0.2.0) ──────────────────────────────
// Responds to messages from origins whitelisted in manifest
// `externally_connectable.matches`. Chrome enforces origin check at IPC
// layer, so any message reaching this handler is already from an
// approved origin.

import { getSavedSessions, getSession } from '../shared/storage.js';
import { generateJmx } from '../shared/jmx-generator.js';
import { generateHar } from '../shared/har-export.js';
import { PULL_API_VERSION, SUPPORTED_FORMATS } from '../shared/constants.js';

const MAX_PAYLOAD_BYTES = 60 * 1024 * 1024; // 60MB — under Chrome's ~64MB IPC cap

chrome.runtime.onMessageExternal.addListener((message, sender, sendResponse) => {
  // Validate message shape
  if (!message || typeof message.action !== 'string') {
    sendResponse({ error: 'invalid_request', message: 'Message must have an action field.' });
    return false;
  }

  // Dispatch — each handler is async, wrap in IIFE and return true to
  // keep the message channel open for async sendResponse.
  (async () => {
    try {
      switch (message.action) {
        case 'ping':
          sendResponse({
            installed: true,
            version: PULL_API_VERSION,
            supportedFormats: SUPPORTED_FORMATS,
            supportedJmxModes: ['fragment', 'standalone'],
          });
          break;

        case 'listRecordings':
          sendResponse(await handleListRecordings());
          break;

        case 'getRecording':
          sendResponse(await handleGetRecording(message));
          break;

        default:
          sendResponse({ error: 'invalid_request', message: `Unknown action: ${message.action}` });
      }
    } catch (err) {
      console.error('[pull-api] handler error:', err);
      sendResponse({
        error: 'generator_error',
        message: err.message || String(err),
      });
    }
  })();

  return true; // keep channel open for async response
});

async function handleListRecordings() {
  const sessions = await getSavedSessions();
  // Also include the current session if it's ended (post-Stop, pre-clear)
  const current = await getSession();
  const all = [];
  if (current && current.endTime) {
    all.push(current);
  }
  all.push(...sessions.map(s => s.session || s));

  return {
    recordings: all.map(s => ({
      id: s.id,
      name: s.name || s.journeyCode || 'Untitled',
      journeyCode: s.journeyCode,
      createdAt: s.startTime,
      endedAt: s.endTime,
      transactionCount: s.transactions?.length || 0,
      requestCount: (s.transactions || []).reduce((n, t) => n + (t.requests?.length || 0), 0),
      correlationCount: s.correlations?.length || 0,
      dataFieldCount: s.dataRequirements?.length || 0,
      sizeBytes: estimateSessionSize(s),
    })),
  };
}

async function handleGetRecording(message) {
  const { id, format, options = {} } = message;

  if (!id) return { error: 'invalid_request', message: 'id is required' };
  if (!format) return { error: 'invalid_request', message: 'format is required' };
  if (!SUPPORTED_FORMATS.includes(format)) {
    return { error: 'invalid_format', message: `Format must be one of: ${SUPPORTED_FORMATS.join(', ')}` };
  }

  const session = await findSessionById(id);
  if (!session) return { error: 'not_found', message: `No recording with id: ${id}` };

  // Generate content per format
  let content, filename, contentType;
  const code = session.journeyCode || 'recording';

  switch (format) {
    case 'json':
      content = JSON.stringify(session, null, 2);
      filename = `${code}_recording.json`;
      contentType = 'application/json';
      break;

    case 'jmx': {
      const mode = options.mode || 'fragment';
      if (!['fragment', 'standalone'].includes(mode)) {
        return { error: 'invalid_options', message: `Invalid JMX mode: ${mode}` };
      }
      content = generateJmx(session, { mode });
      const suffix = mode === 'fragment' ? '' : '_standalone';
      filename = `${code}${suffix}.jmx`;
      contentType = 'application/octet-stream';
      break;
    }

    case 'har':
      content = JSON.stringify(generateHar(session), null, 2);
      filename = `${code}_recording.har`;
      contentType = 'application/json';
      break;

    case 'csv': {
      const dataReqs = session.dataRequirements || [];
      if (dataReqs.length === 0) {
        return { error: 'no_data_fields', message: 'No form fields detected in recording; CSV unavailable.' };
      }
      const headers = dataReqs.map(d => d.suggestedCsvColumn);
      const values = dataReqs.map(d => csvEscape(d.sampleValue || ''));
      content = headers.join(',') + '\n' + values.join(',') + '\n';
      filename = `${code}_test_data.csv`;
      contentType = 'text/csv';
      break;
    }
  }

  // Size guard — Chrome's IPC has a ~64MB structured clone limit
  const payloadBytes = new Blob([content]).size;
  if (payloadBytes > MAX_PAYLOAD_BYTES) {
    return {
      error: 'too_large',
      message: `Recording is ${Math.round(payloadBytes / 1024 / 1024)}MB, exceeds ${MAX_PAYLOAD_BYTES / 1024 / 1024}MB IPC limit.`,
    };
  }

  return { format, filename, contentType, content };
}

async function findSessionById(id) {
  const current = await getSession();
  if (current && current.id === id) return current;

  const sessions = await getSavedSessions();
  const wrapped = sessions.find(s => (s.session || s).id === id);
  return wrapped ? (wrapped.session || wrapped) : null;
}

function estimateSessionSize(session) {
  // Lightweight estimate without re-serialising — sum of known large fields
  let size = 0;
  for (const pr of session.pageResponses || []) {
    size += (pr.body?.length || 0);
  }
  for (const tx of session.transactions || []) {
    for (const r of tx.requests || []) {
      size += (r.requestBody?.length || 0) + (r.responseBody?.length || 0);
    }
  }
  return size;
}

function csvEscape(val) {
  const s = String(val);
  if (s.includes(',') || s.includes('"') || s.includes('\n')) {
    return '"' + s.replace(/"/g, '""') + '"';
  }
  return s;
}
```

## 4. Storage helpers

Add to `shared/storage.js`:

```javascript
// Already exist: getSavedSessions(), getSession()
// No new helpers strictly needed — the handler above uses existing ones.
// If you prefer a single accessor for "all recordings including current":
export async function listAllRecordings() {
  const saved = await getSavedSessions();
  const current = await getSession();
  const all = saved.map(s => s.session || s);
  if (current && current.endTime) all.unshift(current);
  return all;
}
```

Use of `listAllRecordings()` is optional — the `handleListRecordings` in section 3 inlines the same logic. Pick one and use consistently.

## 6. Error handling principles

Follow the pattern in section 3:

- **Never throw unhandled.** Every code path in the handler must call `sendResponse(...)` exactly once. An unhandled throw in an async IIFE causes the external caller to hang until timeout.
- **Always include `error` code and `message`.** The external tool uses the code programmatically; the message is for human debugging / logging.
- **Don't leak stack traces in production.** The catch-all in the top-level dispatcher logs the error to `console.error` but only sends back `err.message` (which is normally short and safe). If this gets stack-tracey in practice, sanitise further.
- **No "fail silently" paths.** Every branch returns either a success response or an error. Never `return undefined` or `return true` without calling `sendResponse`.

## 7. Size / payload considerations

Chrome's `chrome.runtime.sendMessage` uses **structured clone IPC** for payloads. Practical upper bound observed in the wild is ~64MB per response. We set our cap at 60MB to leave headroom.

Format payload rough estimates (for a typical 10-transaction recording with full response capture):

| Format | Typical size | Notes |
|---|---|---|
| `json` | 1–10 MB | Dominant factor is `pageResponses[].body` (full HTML). Can hit cap on 50+ page recordings. |
| `jmx` | 50–500 KB | Extractors, assertions, samplers. Small. |
| `har` | 500 KB – 5 MB | Includes request/response data but not full page HTML. |
| `csv` | 1–10 KB | Trivial — two rows per recording. |

If `json` hits the cap, the tool should fall back to pulling `jmx` or `har` (which are much smaller). Future enhancement: chunked response API (v0.3.0+).

const path = require('path');

const EXT_PATH = path.resolve(__dirname, '..');

test.describe('Pull API', () => {
  let context;

## 10. Out of scope for v0.2.0

Parked for v0.3.0+:
- Chunked response streaming for >60MB JSON payloads.
- `deleteRecording(id)` action for consumer-initiated cleanup.
- `subscribeToRecordings` push notification when a recording is stopped.
- Content hash for integrity verification.
- Additional output formats (Taurus YAML, Gatling DSL, Locust Python).
- Native messaging host for CLI tool integration.

None of these block v0.2.0 shipping. They're good v1 → v2 follow-ups when real usage reveals which matters most.

2. Add `externally_connectable` to manifest.
3. Add bare `ping` handler in service worker. Test from a consuming page that Studio's origin can message.

6. Add `format: "jmx"` branch.

9. Add Playwright tests for each action.
10. Document real extension ID in `docs/RECORDER-PULL-API.md`.

Each step is independently shippable as a dev build — you can dogfood with your own consuming tool along the way.

## 12. Open design decisions

Resolve before or during implementation — none block starting:

- **Sort order of `listRecordings`.** Newest first (most useful for "import my last recording")? Oldest first (stable iteration)? Propose newest first, document in API doc.
- **Default JMX mode.** The popup defaults to `fragment`. Should API default match, or be `standalone`? (Currently spec'd as `fragment` default.)
- **How to handle in-progress recording.** If user calls `getRecording(id)` for a recording that's still being captured (`endTime` not set), respond `{error: "in_progress"}` or return partial data? Propose error.
- **Extension disabled vs not installed.** Both look the same from the consumer's side (`chrome.runtime.lastError`). Do we want `ping` to return different info when disabled? Probably not — the consumer can't tell the difference, and Chrome handles it.
